# Agents

Agent instructions, production-agent notes, and automation prompts for the lAIdies Agent Council.

V0 is recommend-only. Agents can review, score, and propose changes, but they cannot publish, rewrite live website pages, remove features, or change public positioning automatically. Ali remains CEO, editor, and creative director.

## Core Documents

- `agent-command-center.html`: start here for the visual command center, run order, and reusable Codex prompt.
- `agent-organization-map.html`: visual dashboard of the full agent organization.
- `agent-organization-map.md`: Mermaid diagram of the full agent organization.
- `whole-site-agent-council-review.md`: full-site review template; run this before isolated section fixes.
- `fun-and-games-pilot-review.md`: Fun & Games stress test inside the whole-site review.
- `agent-council-operating-system.md`: council purpose, weekly cadence, approval flow, and autonomy limits.

- `ceo-feedback-quality-standard.md`: Ali's high-detail CEO review bar, UX journey rules, and AI credibility proof standard.
- `ai-agent-organization-design-principles.md`: best-practice design principles for the agent organization.
- `ai-risk-register.md`: owned risk register with controls, evidence, residual risk, and review dates.
- `ai-feature-system-card-template.md`: system-card template for AI-assisted or AI-like features.
- `release-safety-case-template.md`: evidence-backed pass/block template for launches.

## Community Digest Rule

The community digest agent should classify chat-room activity into answered/useful, tips, unanswered, repeated themes, and needs-receipts queues. It must not publish sensitive or factual guidance as "correct" without human review.
