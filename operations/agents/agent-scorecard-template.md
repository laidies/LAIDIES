# Agent Scorecard Template

Use this rubric for each agent recommendation or feature review.

Also apply `operations/agents/ceo-feedback-quality-standard.md`. Ali should not be the first person to catch missing user flows, persistence gaps, naming mistakes, wrong brand treatment, weak visuals, fake interactions, or 24-issue scalability problems.

## Employee-Owner Standard

Each agent is responsible for the quality of its assigned surface. A review is not complete unless it identifies the reader job, current weakness, exact recommended fix, evidence, and dependency.

Do not accept vague notes such as "needs polish," "make it more fun," or "improve UX" without the concrete implementation recommendation.

Each website-facing agent should compete for "Best Website Section." The section does not need to be the biggest or flashiest section. It does need to feel intentional, useful, polished, functional, memorable, and unmistakably lAIdies.

## Launch Gate

Choose one:

- PASS: ready to represent lAIdies publicly on desktop and mobile
- IMPROVE: usable, but the agent has a specific upgrade recommendation
- BLOCK LAUNCH: public-facing defect makes the section look broken, chaotic, misleading, unreleased, unsafe, or professionally risky
- INCOMPLETE - NEEDS QA: the agent has not checked the actual page state with enough evidence

Mandatory `BLOCK LAUNCH` triggers:

- visual chaos, awkward blank space, overlap, hidden text, or unfinished layout
- panel/console/modal/drawer open by accident or persistent from URL/hash state
- button/control promises an action that does not visibly happen
- fake spin, shuffle, draw, quiz, signup, form, link, or playlist behavior
- scheduled issue, quiz, card pack, or release language appears public too early
- desktop or mobile behavior has not been checked
- fun references make smart professional women feel talked down to
- user journey gaps: first-time, returning, late-reader, guest, signed-in, completion, empty/error, desktop, or mobile flow not accounted for
- persistence gaps: rewards, cards, stickers, member-card state, preferences, or progress are implied to save without explaining browser-only storage, login, export/import, or sync
- archive gaps: a reader joining weeks later cannot find Issue 01 and released back issues
- brand/detail gaps: wrong issue title, wrong logo/text treatment, inconsistent naming, or placeholder copy reaches review

## Best Section Verdict

- Could this section win best website section this week?
- What makes it a contender?
- What one change would most improve its chance to win?
- What should be protected because it is already working?

If the section cannot compete, the agent must say why plainly and propose the strongest exact fix.

## Scoring Scale

- 1: weak, unclear, risky, or not currently useful
- 2: has potential but needs material work
- 3: acceptable baseline
- 4: strong and worth improving
- 5: excellent, distinctive, and worth protecting or expanding

## Feature Status

Choose one:

- Keep
- Improve
- Simplify
- Hide
- Cut

No feature gets cut from one bad week. Use Hide or Simplify when the issue is timing, clarity, or maintenance burden rather than fundamental fit.
